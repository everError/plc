## 2. memory-structure.md

### 디바이스 메모리 개요

PLC는 데이터를 **디바이스(Device)**라는 메모리 영역에 저장합니다. 디바이스는 크게 **비트 디바이스**와 **워드 디바이스**로 나뉩니다.

### 비트 디바이스 (1비트 단위)

| 디바이스 | 이름        | 용도                       |
| -------- | ----------- | -------------------------- |
| X        | 입력 릴레이 | 외부 입력 신호 (읽기 전용) |
| Y        | 출력 릴레이 | 외부 출력 신호             |
| M        | 내부 릴레이 | 내부 플래그, 임시 저장     |
| L        | 래치 릴레이 | 정전 유지 (래치)           |
| B        | 링크 릴레이 | 네트워크 공유용            |

### 워드 디바이스 (16비트 단위)

| 디바이스 | 이름            | 용도                       |
| -------- | --------------- | -------------------------- |
| D        | 데이터 레지스터 | 수치 저장 (가장 많이 사용) |
| W        | 링크 레지스터   | 네트워크 공유용            |
| R        | 파일 레지스터   | 대용량 데이터 저장         |
| TN       | 타이머 현재값   | 타이머 경과 시간           |
| CN       | 카운터 현재값   | 카운터 현재 카운트         |

### 주소 체계

| 디바이스   | 주소 형식 | 예시          |
| ---------- | --------- | ------------- |
| D, M, L, R | 10진수    | D100, M0, L50 |
| W, B       | 16진수    | W1A0, B3F     |

### 비트 ↔ 워드 관계

워드 디바이스도 비트 단위 접근 가능합니다.

```
D100 (16비트)
├── D100.0  (bit 0)
├── D100.1  (bit 1)
│   ...
└── D100.F  (bit 15)
```

비트 디바이스도 워드 단위로 묶어서 읽을 수 있습니다.

```
M0 ~ M15 → 1워드로 읽기 가능
```
